easyValidator

一个简单·强大的jquery表单验证插件

easyValidator是一个jquery表单验证插件，力求把复杂繁琐的表单验证工作简化,同时又具有较大的适用性和灵活性，能够应付千变万化的各种不同形式的表单验证。

主要特征：

  • 可自定义验证的数据类型
  • 可自定义验证的提示方式
  • 支持ajax远程验证
  • 直接在表单元素上以添加属性的方式进行配置，简单明了
  • 兼容IE6+、chrome、firefox、opera等各大主流浏览器

使用方法：

首先要在页面引入jquery与easyValid
?
<script src="jquery-1.8.3.min.js"></script>
<script src="easyvalid-1.0.min.js"></script>
然后在dom加载完毕后，或者在要验证的表单html代码后面调用easyValid函数
?
<script>
    $(form_selector).easyValid(opts);
</script>
其中form_selector为要进行验证的表单的选择器，opts为本插件的配置对象。
例如，有一段表单的html代码:
?
<form id="theform">
    用户名: <input type="text" name="username" datatype="w6~12" />
    密　码：<input type="password" name="password" datatype="w6~20" />
    邮　箱：<input type="text" name="email" datatype="email" />
    <input type="submit" value="提交" />
</form>
然后我们对应的调用easyValid插件进行验证的代码就是：
?
<script>
   $('#theform').easyValid();
</script>
如上所示，我们甚至不需要给easyValid函数传入任何配置参数，插件就能正常工作，此时使用的都是插件默认的配置。

配置参数：

配置参数既可以html属性的方式写在html标签上，也可以js对象直接量的的方式传入easyValid函数。比如：
?
<form id="theform">
   <input type="text" name="email" errmsg="填写错误" />
</form>
<script>
   $('#theform').easyValid({
      errmsg:'填写错误'
   });
</script>
如果同一个参数在两种方式中都进行了配置，例如上面的例子，则以在html标签中的为准。
在html标签中配置的参数主要是单独针对该html标签代表的表单元素的，而在easyValid函数中传入的配置参数则会被所有表单元素共用。
下面是对所有参数的一个说明：
属性类型默认值说明
datatype String|Function 无默认值 验证类型。表单元素的值必须符合该验证类型才能通过验证。
----------------------------------------------------
系统提供了一些内置的常用验证类型，如下：
* : 不为空格的任何字符
w : 字母、数字和下划线
d : 数字(不包括小数)
en : 英文字母，大小写都包括
zh : 中文字符
以上几种验证类型还可以自定义长度，例如:
d5:代表必须是5个数字
d5~10:代表5至10个数字
d5~:代表至少5个数字
zh4:代表必须是4个中文字符
上面介绍的几种类型都可以这样用
此外，还有几种固定的验证类型，如下：
cellphone : 手机号码
telphone : 固定电话
email : 电子邮箱
idcard : 身份证号码
----------------------------------------------------
如果以上无法满足你的需求，还可以直接写正则表达式
例如：<input type="text" datatype="/^\d{5}$/" /> 看到了吧，开头和结尾的这两个 "/" 斜杠符号即表明你使用的是自定义的正则表达式，这很好理解，我们创建正则表达式直接量的时候也是这种形式
----------------------------------------------------
如果自定义的正则表达式还是不能满足你的需要，那还可以使用自定义函数来验证
例如：<input type="text" datatype="notNull()" /> 这里我们使用的自定义验证函数是notNull,同时看到后面有一对括号，这就表明使用的是自定义的函数验证
自定义验证函数必须放在插件的自定义函数集合里，如果在这个集合里没有找到该函数，则任何值都会通过验证。
系统会给自定义验证函数传入两个参数，第一个是当前验证的表单的元素的值，第二个是当前验证的表单的元素的配置对象
自定义验证函数必须返回相应的代码，来表明验证的情况:
返回数字0或者不返回任何值表示验证通过；返回null时表示可以为空；返回数字1表示验证未通过；
----------------------------------------------------
此外，还有几种特殊的验证类型
re:name : 一致性验证,使用时name替换成具体的name值,表示值必须与当前表单中name属性为name的元素的值相同，才能通过验证，用于两次输入密码的一致性验证
tiptype Number|Function 2 表单验证的提示类型
----------------------------------------------------
程序内置了两种方式,是以数字来表示的
1 : 最简单的alert弹出提示
2 : 灵活、可自定义样式的提示,也是本插件默认的类型。当使用这种提示类型时，程序会在当前验证表单元素的同级元素中查找class名为该表单元素配置对象中domcls属性的值的元素，如果找不到这样的元素，则程序会在表单元素后面自动创建一个span元素，类名为配置对象中domcls属性的值。这个元素即是进行提示的UI元素，我们可以通过css来自定义它的展现形式。
----------------------------------------------------
此外还可以使用自定义的函数来决定你想要的提示方式，达到随心所欲的灵活度
比如: <input type="text" datatype="email" tiptype="emailUI()" /> 像datatype使用自定义函数时一样，要以一对小括号来标示，表明使用的是自定义函数，同时该函数必须存在于插件配置对象中的fn属性所定义的插件自定义函数集合中。
我们在定义这个自定义提示函数时可以使用三个参数：第一个为当前验证的状态码;第二个为当前验证的表单元素;第三个为该表单元素的配置对象。我们可以根据这些参数来创造出你想要的任何一种提示UI
domcls String|Function ev-tip-dom 当tiptype为2时此参数才有用，用来指定提示元素的类名，程序会根据此类名在表单元素的的同级元素中进行查找，如果找不到，则会在表单元素后面以该属性的值为类名创建一个元素
----------------------------------------------------
如果你的提示元素的位置很特殊，不是在表单元素的同级元素中，那么该值同样可以使用自定义函数
例如: <input type="text" domcls="getTipDom()" /> 使用自定义函数时，要以一对小括号来标示，表明使用的是自定义函数，同时该函数必须存在于插件配置对象中的fn属性所定义的插件自定义函数集合中。
我们在定义这个自定义函数时可以使用两个参数：第一个是当前的表单元素(为jquery对象);第二个是该表单元素所在的form元素
该函数必须返回一个元素，作为进行提示的UI元素，同时程序会自动给这个元素加上domcls属性指定的类名。
errcls String ev-err 验证未通过时提示元素的类名，即验证未通过时程序会自动给提示元素加上这个类名。
emptycls String 同errcls的值 值为空时提示元素的类名，即当前验证的表单元素值为空时程序会自动给提示元素加上这个类名，如果未特别指定该值，则与errcls相同。
okcls String ev-ok 验证通过时提示元素的类名，即验证通过后程序会自动给提示元素加上这个类名。
tipcls String ev-tip 表单元素获得焦点时提示元素的类名，即当表单元素获得焦点时程序会自动给提示元素加上这个类名。
waitcls String ev-wait 当正在进行ajax验证时提示元素的类名，即在发送ajax验证请求并等待返回结果的这段时间，程序会自动给提示元素加上这个类名。
f_errcls String evf-err 验证未通过时表单元素的类名，即验证未通过时程序会自动给当前验证的表单元素加上这个类名。
f_emptycls String 同f_errcls的值 值为空时表单元素的类名，即当前验证的表单元素值为空时程序会自动给这个表单元素加上这个类名，如果未特别指定该值，则与f_errcls相同。
f_okcls String evf-ok 验证通过时表单元素的类名，即验证通过后程序会自动给当前验证的表单元素加上这个类名。
f_tipcls String evf-tip 表单元素获得焦点时表单元素的类名，即当表单元素获得焦点时程序会自动给这个表单元素加上这个类名。
f_waitcls String evf-wait 当正在进行ajax验证时表单元素的类名，即在发送ajax验证请求并等待返回结果的这段时间，程序会自动给当前验证的表单元素加上这个类名。
errmsg String 填写错误 验证未通过时的提示信息
okmsg String 空 验证通过时的提示信息
tipmsg String 请填写 表单元素获得焦点时的提示信息
waitmsg String 正在验证... 进行ajax验证时的提示信息
emptymsg String 不能为空 不能为空时的提示信息
textcls String 无默认值 显示提示信息的容器的类名，为空时则直接显示在提示元素里面，如果设置了该属性，则会在提示元素里查找类名为该属性的元素作为存放提示信息的容器，该属性是考虑到有些提示元素可能有多层的结构
empty Boolean false 是否允许为空，如果为true则表示允许为空，当允许为空后，空值也可以通过验证，并且不会进行提示
ajaxurl String 空 ajax验证地址。如果指定了该属性的值，则插件会向这个值表示的地址发送一个ajax请求，同时当前验证表单元素的值也会以ajaxname属性指定的参数名随之发送。后台的ajax验证程序必须返回一个json对象，该对象有两个属性:errno和errstr. errno为验证结果的标识码，只有errno的值为0时才代表通过验证，其他值都表示未通过验证,errstr为错误信息
ajaxname String ajaxdata 进行ajax验证时发送数据的参数名，如果为空则插件会把当前验证的表单元素的name值作为该参数名，如果name值也为空，则会使用默认值ajaxdata
onvalid Function function(){} 进行表单元素的验证后触发的回调函数，该函数可以使用三个参数,第一个为当前验证的表单元素,第二个为验证的结果代码,第三个为当前验证表单元素的配置对象

下面还有几个与插件行为有关的参数，他们只可以在easyValid函数中进行配置
属性类型默认值说明
all Boolean true 表单提交时是否验证所有表单元素,如果为false，则遇到未通过验证的就会停止后面元素的验证
focus Boolean true 如果为true，则提交表单时未通过验证的第一个表单元素自动获得焦点
fn Object {} 存放插件所要使用的自定义函数的集合，有些属性需要用到函数时则那些函数都必须放在这个里面
例如:
 
            $('#foo').easyValid({
                fn:{
                        fn1:function(){...},
                        fn2:function(){...},
                        fn3:function(){...},
                        fn4:function(){...}
                        ...
                    }
            });
